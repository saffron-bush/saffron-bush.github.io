There are 3 parts to a 'for Loop

  • The initial expression - let 1 = 0>
    It sets the loop control variable to an initial value (because the list is an array, we enter the number [0] to begin the loop at the toothbrush)
  • The conditional expression - i < packingList.length

Arrays, objects and functions all use reference values. But what are reference values and what do they have to do with cooking? Itis all to do with how variables are passed through a statement and how the computer stores them within it's memory. Sounds complicated right? Well let's use a cake analogy to explain how information flows between these items.

Sisters are the best and can be super helpful when you need advice, help or information. Let's say your sister walks in the room with a delicious looking piece of cake. You ask your sister, where did you get that piece of cake?

**A variable with value type (aka a primitive) would store information like this:**

My sister would go to the kitchen, collect a slice of cake from the cupboard, and hand-deliver the slice of cake to me. This is passing by value type. Very kind, but it is static information. I am just asking for cake and it appears in my hand. Examples of value types are: strings, numbers, booleans

**A variable with a reference type would store information like this:**

She would say to me "yeah, there's some in a plastic container in the kitchen, middle shelf on the right." she hasn't given me any cake, but told me where the cake slices are located, so now I can go and get one on my own. This is passing by reference. This means I am learning where the variable (cake) is and how to locate it, if I ever want to come back to it or change the location of it.

Unlike value types, a reference type doesn't store its value directly. Instead, it stores the address where the value is being stored. In other words, a reference type contains a pointer to another memory location that holds the data. A reference value allows you to add, change, or delete properties at any time.

Now to continue with the cake analogy, objects are the most powerful data type of javascript and are used in almost everything. Functions are objects, Arrays are objects and objects are objects. Objects store data in key value pairs, the keys can allow you to add, change or delete properties at any time and can be accessed in any order. We can either use a dot followed by key name obj.name or a square bracket along with key in a string format obj['name'] to access the value.

Now I already mentioned that functions are also an object. But what is a function? A function is a set of instructions to perform a task and return it. Values can be passed into a function, and the function will return a value. Functions are first-class objects, because they can have properties and methods just like any other object. Functions can be passed by primitive values or by reference values. If the function reassigns a primitive value parameter, the value won't change outside of the function. However, if the function uses an object type parameter, the changes will impact the outside of the function.
